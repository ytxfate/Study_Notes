
1. 使用 `ss` 或者 `netstat` 命令过滤端口信息, 输出内容右边 `pid=70379` 即是进程号, 如果看不到进程信息说明当前用户没有权限, 需要使用更高权限的用户或 `sudo` 来执行 `ss` 命令
```bash
> sudo ss -ntlp | grep 9998
LISTEN 0      511               0.0.0.0:9998       0.0.0.0:*    users:(("nginx",pid=70383,fd=6),("nginx",pid=70379,fd=6))
```
2. 通过进程号查询项目执行目录, `/proc/70379/` 里的 `70379` 即是上一步中查询到的进程号, `cwd` 为进程执行目录, `exe`为进程可执行文件的路径, 如果看不到 `->` 后的内容说明当前用户没有权限, 需要使用更高权限的用户或 `sudo` 来执行 `ls` 命令
```bash
> sudo ls -alh /proc/70379/ | egrep 'cwd|exe'
lrwxrwxrwx   1 xxx xxx 0  2月26日 11:19 cwd -> /usr/local/nginx
lrwxrwxrwx   1 xxx xxx 0  2月26日 11:18 exe -> /usr/local/nginx/sbin/nginx
```
3. 有可能步骤1中找到的进程不是父进程, 需要结合 `ps -ef` 命令查询父进程号再执行步骤2
